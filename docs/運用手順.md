# NotebookLM 研修復習ポータル 公開と運用手順

このドキュメントは、研修用ポータルサイトの運用・更新方法を説明します。

## 1. ポータルサイトの公開方法 (GitHub Pages)

全世界からアクセスできるURL（例: `https://juyeongjung1.github.io/notebookLM_View_Portal/`）を発行する手順です。

1. [GitHubリポジトリ画面](https://github.com/juyeongjung1/notebookLM_View_Portal)を開きます。
2. 画面上部にある歯車アイコンの **Settings**（設定）をクリックします。
3. 左側のメニューから **Pages** をクリックします。
4. **Build and deployment** の項目にある **Source** が `Deploy from a branch` になっていることを確認します。
5. **Branch** の下のプルダウンで `main` を選択し、右側の `/(root)`のままにして、 **Save** ボタンを押します。
6. 1〜2分後、同じ画面の上部に `Your site is published at ...` と表示されたURLが、受講者に配るURLになります。

## 2. リンクの追加・更新方法

新しくNotebookLMの動画や音声を作った場合の更新手順です。

### 準備: NotebookLMでの操作
1. NotebookLMの公開したい章・ノートブックを開きます。
2. 画面右上の「共有設定」を開き、「**リンクを知っている全員**」がアクセスできるように設定して、リンク（URL）をコピーします。

### 更新: ポータルサイトへの反映
1. 手元のPCで、作成したポータルサイトのフォルダ（`c:\work\notebookLM`）にある **`data.js`** ファイルを開きます。
2. 以下のような形式で、新しい項目を `[` と `]` の間（または末尾の `,` 後）に追加します。
   ```javascript
   {
     id: 7, // 連番
     category: "IT基礎", // 章やカテゴリ名
     title: "IT基礎-3章（解説動画）",
     type: "video", // 動画なら"video", 音声なら"audio"
     description: "ここに動画の簡単な説明文を書きます。",
     url: "https://notebooklm.google.com/..." // ←ここにコピーしたURLを貼り付ける
   }
   ```
3. 保存したら、VSCodeやターミナルでコミットとプッシュを行います。
   ```bash
   git add data.js
   git commit -m "新しい章の動画を追加"
   git push origin main
   ```
   これで、数分後に自動的に公開サイトにも新しいボタンが追加されます！

## 3. カテゴリ（章）のタブについて
`data.js` の `category` に新しい名前（例: `"特別講義"`）を設定するだけで、画面上の「タブ」は自動的に新しく生成されます。
特別な設定は必要ありません。
